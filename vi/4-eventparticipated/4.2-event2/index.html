Event 2

Bài Thu Hoạch “DevOps on AWS”

Mục đích của sự kiện

  • Hiểu tổng quan về DevOps và văn hóa DevOps trong doanh nghiệp.
  • Nắm được các khái niệm CI/CD, IaC và các công cụ DevOps phổ biến.
  • Tìm hiểu cách triển khai DevOps trên AWS: CloudFormation, CDK, Amplify.
  • Học kinh nghiệm thực tế từ các diễn giả trong cộng đồng AWS.

Danh sách diễn giả

  • Trương Quang Tịnh – AWS Community Builder, Platform Engineer – Timex
  • Hoàng Kha – DevOps Engineer
  • Nguyễn Bảo – Cloud Engineer
  • Nguyễn Thịnh – AWS Developer / CDK Specialist

Nội dung nổi bật

1. Building a DevOps Culture

@Speaker: Trương Quang Tịnh

Dev – DevOps – Operations

  • Dev: viết code, phát triển tính năng.
  • Operations: vận hành, triển khai, giám sát hệ thống.
  • DevOps: hợp nhất Dev và Ops → tăng tốc độ triển khai + giảm lỗi.

Các yếu tố cốt lõi của DevOps

  • Collaboration & Shared Responsibility: cùng chịu trách nhiệm về sản phẩm.
  • Automate Everything: tự động hóa tối đa (test, deploy, infra…).
  • Continuous Learning & Experimentation: học liên tục, thử nghiệm.
  • Measurement: đo lường bằng metrics rõ ràng.

Next Generation DevOps

  • DevOps
  • Cloud
  • Platform Engineering
  • Site Reliability Engineering (SRE)

DevOps Success Metrics

  • Monitoring deployment health
  • Improve agility
  • Speed of delivery
  • Reduce failures

2. DevOps Practices & CI/CD Pipeline

@Speaker: Hoàng Kha

CI/CD Pipeline

  • Giới thiệu quy trình CI/CD chuẩn.
  • Triển khai pipeline với GitHub Workflow.

Nội dung chính

  • Tối ưu chi phí DevOps.
  • Áp dụng open-source trong pipeline.
  • DevOps practice trong thực tế.

Continuous Integration (CI)

  • Quản lý repo, tổ chức workflow của team dev.
  • Testing ở mọi giai đoạn.
  • Build có thể bị fail vì:
    • Code lỗi
    • Missing dependency
    • Test fail
    • Conflict khi merge

Continuous Delivery (CD)

  • Tự động triển khai từ staging → production.

Best Practices

  • Branching & merging chuẩn
  • Pull request workflow
  • Quy tắc đặt tên branch
  • Tổ chức source code
  • Module hóa container

3. Infrastructure as Code (IaC)

@Speaker: Nguyễn Bảo

IaC là gì?

  • “ClickOps” (tạo thủ công trên console) → không hiệu quả, dễ sai.
  • IaC: sử dụng code để quản lý hạ tầng.

AWS CloudFormation

  • Công cụ IaC chính thức của AWS.
  • Stack: nhóm tài nguyên được định nghĩa trong 1 file.
  • Template: file YAML mô tả resource.

CloudFormation Workflow

  • Template → Deploy → Stack
  • Template có thể lưu trong Git Repo để versioning.

Configuration Drift

  • Khi hạ tầng thực tế khác với hạ tầng trong template.
  • CloudFormation hỗ trợ Drift Detection để kiểm tra sai lệch.

4. AWS CDK (Cloud Development Kit)

@Speaker: Nguyễn Thịnh

CDK Components

  • Construct: khối xây dựng cơ bản.

    • L1: giống CloudFormation (raw).
    • L2: có cấu hình mặc định, dễ hiểu.
    • L3: pattern hoàn chỉnh (ví dụ: API + Lambda + DynamoDB).

  • App: root construct.

  • Stack: chứa các resource.

Ưu điểm CDK

  • Viết hạ tầng bằng ngôn ngữ lập trình (TypeScript, Python…).
  • Dễ đọc, dễ tái sử dụng.

AWS Amplify

  • Hoạt động dựa trên CloudFormation + CDK.
  • Mỗi môi trường (production, sandbox) là một stack riêng biệt.

Lưu ý

  • Tránh circular dependency khi dùng SDK trong cùng template.

5. Choosing Between IaC Tools

@Speaker: Nguyễn Bảo

So sánh các công cụ IaC

  • Terraform
  • OpenTofu
  • Pulumi
  • Mỗi tool có ưu nhược riêng → tùy vào nhu cầu dự án.

Điều học được từ sự kiện

Kiến thức & Tư duy DevOps

  • Hiểu rõ văn hóa DevOps và cách áp dụng trong tổ chức.
  • Nắm quy trình CI/CD chuẩn và workflow chuyên nghiệp.
  • Phân biệt IaC tools và chọn lựa tùy theo use-case.

Kiến thức AWS

  • Sử dụng CloudFormation, CDK để quản lý hạ tầng.
  • Hiểu cơ chế stack, template và drift detection.
  • Biết cách dùng Amplify để triển khai frontend.

Kinh nghiệm thực tế

  • Tối ưu chi phí DevOps pipelines.
  • Áp dụng open-source để giảm cost.
  • Sử dụng best practices khi tổ chức source code & repo.

Bài học rút ra

  • IaC là bắt buộc để scale hệ thống.
  • Automation là chìa khóa của DevOps hiện đại.
  • DevOps ≠ tool → là văn hóa & tư duy làm việc.
  • CDK giúp hạ tầng dễ quản lý, dễ tái sử dụng, dễ mở rộng.
